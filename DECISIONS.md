Architektura oparta na DDD-lite. Warstwa domeny (Entity) zawiera logikę biznesową: soft delete, walidacje. Warstwa aplikacji (ProductService) - orkiestracja, eventy, reguły. Repository - Doctrine ORM. Controller - tylko REST, zero logiki biznesowej. Optimistic locking rozwiązuje konflikty edycji. UUID zamiast auto-increment. Decimal(10,2) na ceny - precyzja finansowa. PriceHistory jako osobna encja - lepsza normalizacja. Event ProductPriceChanged - wymaganie zadania, aktualnie tylko logowanie. SKU unikalne w ramach aktywnych produktów - usunięte mogą mieć duplikaty. Waluty: PLN, EUR, USD.Integracja z API NBP (http://api.nbp.pl) do zaimplementowania.  Brak testów integracyjnych - ograniczenia czasowe.
